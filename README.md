Dieses Repository dient als Übungsprojekt, um den GitHub-Workflow kennenzulernen:
Pull Requests erstellen, Reviews durchführen und Änderungen mergen.

## Erste Schritte

1. Repository klonen
2. Änderungen auf einem neuen Branch vornehmen

4. Review abwarten und mergen
